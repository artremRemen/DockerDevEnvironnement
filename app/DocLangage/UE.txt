nomenclature

    class dériver d'un Actor commence par A 
    class dériver d'un Object commence par U
    class dériver d'un Enums commence par E
    class dériver d'un Interface commence par I
    class dériver d'un Template commence par T
    class dériver d'un Swidget(UI) commence par S     
    le reste des class commence par F

Type de fonction

    UCLASS()        utiliser pour appeller la génération de donnée d'une class (doit etre dériver d'un UObject)

    USTRUCT()       utiliser pour appeller la génération de donnée d'une structuree 

    UInterface()    va créer un lien entre deux Bp qui ne sont en temps normal pas compatibles ensembles

----------

type de proprieter (a l'interieur des fonctions)

    GENERATED_BODY()        UE le remplace par tous ce qui est necessaire pour le générer

    UPROPERTY()             déclarer une variable dans une class ou une structure qui va etre utiliser par une property. il peut autoriser la duplication, serialisation et les access depuis les Blueprints.  ils est aussi utiliser par la "corbeille" pour garder la référence ci c'est un UObjectObject.

    UFUNCTION()             déclarer une class d'une Uclass ou une Ustructure qui va etre utiliser dans une Ufonction. Une Ufonction peut autoriser la class a etre appeller dans une BP 

atrubuts des proprieter

    Blueprintable       peut etre utiliser dans un BP

    BlueprintReadOnly   peut etre lus par un BP mais pas modifier

    EditAnywhere        peut etre modifier dans la fenetre d'édition dans le viewport

    Category            définis la catégorie dans laquel ce trouve les variables/fonctions

    BlueprintCallable   peut etre appeller par un BP
